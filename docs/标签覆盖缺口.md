# 标签覆盖缺口 / Tag Coverage Gaps

- **统计口径**：每道题的 `tags` 字段是"解题方法/思维方法"；知识点的 `tag` 字段是它对应的方法标签。
- **阈值约定**：知识点每个标签应至少 1 篇讲解；题目每个标签建议至少 3 道（便于"使用该方法的例题"列表有梯度）。

## 三、结构性观察

1. **travel（行程）模块暂无专属方法标签**：6 道行程题全部 `tags: []`。要么在 `METHOD_TAGS` 中新增 `travel` 类别（如"相遇追及"、"流水行船"、"环形跑道"、"多次相遇"、"发车间隔"、"火车过桥"），要么明确不为行程题打方法标签、仅用模块区分 —— 需要决策后再补题/改规则。
2. **numberTheory 模块方法标签偏少**：仅 `整除特征 / 同余 / 质因数分解` 三条高频；`余数 / 最大公约与最小公倍 / 完全平方数 / 进位制` 等常见数论方法尚未进入 `METHOD_TAGS`，是否补入待定。
3. **docs/ 与 tags.ts 的对齐**：
   - `docs/几何五大模型.md` 对应 `蝴蝶/鸟头/相似/燕尾/等积变形` —— 已全部有标签与知识点，可继续增题。
   - `docs/排列组合.md` 对应 counting 全集 —— **标签齐全但知识点缺 7 条**（见 §1），应优先补齐。
   - `docs/幻方与数阵图.md` 对应 magicSquare 全集 —— **缺 `比较法 / 试填法` 两个知识点**；比较法尤其应尽快补，因题目里用到 9 次。
   - `docs/统筹规划.md` 中的"最优化/运筹"思想目前没有对应方法标签（只能用 `枚举法 / 分类讨论 / 不变量` 近似）；若要独立展示，需要新增标签。
