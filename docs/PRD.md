# 奥数题库 MVP 产品需求文档

---

## 1. 项目概述

### 1.1 项目背景

- 小学奥数（数学思维训练）市场规模达 48 亿元，年复合增长率 12.3%
- 现有产品多为“大而全”的静态题库，缺乏“深入浅出”的动态讲解能力
- 家长和学生的核心痛点：**“题目有答案，但看不懂推导过程”**

### 1.2 产品定位

一款 **以高质量讲解为核心** 的轻量级奥数思维训练工具。不做大而全的题库堆砌，而是通过 **分步图文 + 多解法对照 + （后续）AI 引导** 的组合，让每一道题都“有人带着想清楚”。

### 1.3 目标用户

| 用户类型        | 核心诉求                                     |
| :-------------- | :------------------------------------------- |
| 3-6 年级小学生  | 有奥数兴趣或培优需求，需要清晰易懂的解题思路 |
| 陪读家长        | 希望有能力辅导孩子，但自身对奥数解法不熟悉   |
| 独立老师/小机构 | 需要高质量的讲解素材辅助教学                 |

---

### 1.4 词汇表

| 术语   | 解释                                     |
| :----- | :--------------------------------------- |
| MVP    | Minimum Viable Product，最小可行产品     |
| 变式题 | 与母题知识点相同、情境或数据不同的衍生题 |

## 2. MVP 范围

### 2.1 MVP 目标

用 **一道经典题** 跑通“价值感知 → 解锁意愿”的完整链路，验证用户是否愿意为“讲得清楚”付费。

### 2.2 MVP 包含功能

| 模块       | 功能点                           | 优先级 | 说明                                        |
| :--------- | :------------------------------- | :----- | :------------------------------------------ |
| 题目展示   | 题干清晰呈现                     | P0     | 包含头数、脚数等关键信息                    |
| 图文解析   | 画图法分步拆解（免费）           | P0     | 作为“价值感知”钩子，完整展示                |
| 多解法解锁 | 假设法 + 方程法（付费/留资解锁） | P0     | 作为“付费意愿”钩子，对照学习                |
| 举一反三   | 同类型变式题练习（解锁后可见）   | P0     | 验证学习效果，属于解锁权益                  |
| 解锁墙     | 点击解锁 → 留资/支付             | P0     | MVP 先用“留手机号/微信”做意愿验证，不接支付 |
| 知识点标签 | 关联知识点归类                   | P1     | 便于后续扩展                                |

### 2.3 MVP 不包含功能

- **视频讲解**（V2 再做，MVP 阶段重资产 ROI 过低）
- 完整用户登录/注册系统（解锁墙用轻量留资即可）
- 真实在线支付（先验证意愿，不验证金额）
- 题库后台管理系统
- AI 对话引导（V3 规划）
- 学习进度追踪与数据看板
- 多端适配（先做移动端 H5）

## 3. 详细功能设计

### 3.1 题目信息模块

- **内容**：题干文字 + 关键数据高亮
- **示例**：
  > 笼子里有鸡和兔，共 **30个头**，**88只脚**。问鸡和兔各有多少只？

### 3.2 图文解析模块

提供三种解法分步拆解，**分层解锁**：

| 解法   | 可见性       | 定位                     |
| :----- | :----------- | :----------------------- |
| 画图法 | 免费全部可见 | 价值钩子，建立讲解信任感 |
| 假设法 | 解锁后可见   | 进阶技巧，速度更快       |
| 方程法 | 解锁后可见   | 高年级衔接，思维跃迁     |

**方法一：画图法**

- ① 画 30 个圆圈代表 30 个头

  ```
  ○ ○ ○ ○ ○ ○ ○ ○ ○ ○
  ○ ○ ○ ○ ○ ○ ○ ○ ○ ○
  ○ ○ ○ ○ ○ ○ ○ ○ ○ ○
  ```

- ② 每个下面画 2 只脚（假设全是鸡）→ 共 60 只脚

  ```
  ○ ○ ○ ○ ○ ○ ○ ○ ○ ○   ← 30 个头
  ∥ ∥ ∥ ∥ ∥ ∥ ∥ ∥ ∥ ∥   ← 每个 2 只脚
  ∥ ∥ ∥ ∥ ∥ ∥ ∥ ∥ ∥ ∥   合计：30 × 2 = 60 只脚
  ∥ ∥ ∥ ∥ ∥ ∥ ∥ ∥ ∥ ∥
  ```

- ③ 实际 88 只，还差 28 只

  ```
  目标  ████████████████████████████████████████████  88
  当前  ████████████████████████████████              60
                                        ↑ 还差 28 只脚
  ```

- ④ 每只鸡补 2 只脚变兔 → 补 14 只（28 ÷ 2 = 14）

  ```
  ○ ○ ○ ○ ○ ○ ○ ○ ○ ○ ○ ○ ○ ○ │ ○ ○ ○ ○ ○ ○ ○ ○ ○ ○ ○ ○ ○ ○ ○ ○
  ∥ ∥ ∥ ∥ ∥ ∥ ∥ ∥ ∥ ∥ ∥ ∥ ∥ ∥ │ ∥ ∥ ∥ ∥ ∥ ∥ ∥ ∥ ∥ ∥ ∥ ∥ ∥ ∥ ∥ ∥
  ╳ ╳ ╳ ╳ ╳ ╳ ╳ ╳ ╳ ╳ ╳ ╳ ╳ ╳ │
  ╳ ╳ ╳ ╳ ╳ ╳ ╳ ╳ ╳ ╳ ╳ ╳ ╳ ╳ │
       🐰 兔（4 脚）× 14        │        🐔 鸡（2 脚）× 16
  ```

- ⑤ 结论：兔 14 只，鸡 16 只

  ```
  🐰×14 + 🐔×16 = 30 只（头）
  4×14 + 2×16 = 56 + 32 = 88 只（脚）✓
  ```

**方法二：假设法**

- ① 假设全是鸡：30 × 2 = 60 只脚
- ② 实际相差：88 - 60 = 28 只脚
- ③ 每只兔比鸡多 2 只脚
- ④ 兔数：28 ÷ 2 = 14 只
- ⑤ 鸡数：30 - 14 = 16 只

**方法三：方程法（高年级适用）**

- ① 设鸡 x 只，兔 y 只
- ② x + y = 30
- ③ 2x + 4y = 88
- ④ 解得：x = 16，y = 14

### 3.3 举一反三模块

- **变式题目**：
  > 停车场上共有汽车和摩托车 28 辆，轮子总数 86 个。汽车 4 轮，摩托车 2 轮，问各有多少辆？
- **交互**：提供输入框，用户提交答案后显示正误判断
- **解锁策略**：与假设法/方程法共用同一道解锁墙，解锁后一并开放

### 3.4 解锁墙模块

- **触发时机**：用户点击“假设法 / 方程法 / 变式题”任一入口
- **交互形式**：弹出浮层，要求留手机号或微信（MVP 阶段不接入支付，先验证意愿）
- **解锁后行为**：本机 `localStorage` 记为已解锁，刷新保留
- **文案建议**：突出“多一种解法 = 多一种思维训练”，而不是硬推付费

### 3.5 知识点标签模块

- 展示本题关联的知识点标签：`#假设法` `#鸡兔同笼` `#四年级` `#方程初步`
- 点击标签可跳转到同类题目列表（V2 实现）

## 4. 技术方案

### 4.1 技术栈选型

| 层级 | 技术                                     | 说明                           |
| :--- | :--------------------------------------- | :----------------------------- |
| 前端 | PNPM + Shadcn + Vite v8 + TailwindCSS v4 | 灵活，可扩展性强               |
| 部署 | Vercel                                   | 免费，自动 CI/CD               |
| 后端 | 暂不需要                                 | MVP 纯静态，数据写死在 JSON 中 |

### 4.2 数据结构示例

```json
{
  "id": 1,
  "title": "鸡兔同笼·基础变式",
  "question": "笼子里有鸡和兔，共30个头，88只脚。问鸡和兔各有多少只？",
  "solutions": {
    "drawing": { "locked": false, "steps": ["步骤1...", "步骤2..."] },
    "hypothesis": { "locked": true, "steps": ["步骤1...", "步骤2..."] },
    "equation": { "locked": true, "steps": ["步骤1...", "步骤2..."] }
  },
  "variant": {
    "question": "停车场上共有汽车和摩托车28辆...",
    "answer": { "car": 15, "motorcycle": 13 }
  },

```

### 4.3 页面结构

```
单题页面 (Mobile H5)
├── Header：题目标题 + 难度标签
├── QuestionCard：题干展示
├── SolutionTabs：图文解析（画图法免费 / 假设法·方程法加锁）
├── PracticeCard：举一反三练习（加锁）
├── UnlockSheet：解锁浮层（留资表单）
└── TagsRow：知识点标签
```

## 5. 验收标准

### 5.1 功能验收

- [ ] 题目信息正确展示
- [ ] 画图法（免费）完整展示，无需解锁
- [ ] 假设法、方程法默认加锁，触发解锁浮层
- [ ] 解锁后刷新仍保持解锁状态（localStorage）
- [ ] 变式题输入答案后可判断正误
- [ ] 知识点标签正确显示
- [ ] 页面在主流手机尺寸下正常显示

### 5.2 体验验收

- [ ] 图文解析步骤分明，易于跟随
- [ ] 解锁浮层文案友好，不强硬推销
- [ ] 页面加载速度 < 2 秒

### 5.3 商业验证指标

| 指标           | 目标值 | 说明                      |
| :------------- | :----- | :------------------------ |
| 画图法读完率   | > 50%  | 滚动到底部 / 总访问       |
| 解锁点击率     | > 20%  | 点击任一解锁入口 / 总访问 |
| 解锁留资完成率 | > 30%  | 成功提交留资 / 解锁点击   |
| 变式题尝试率   | > 60%  | 点击练习 / 解锁用户       |

## 6. 后续迭代规划

### V2（1-2 个月后）

- **引入视频讲解**（3 分钟手绘风格，基于 MVP 验证后的付费意愿决定是否自制 or 外包）
- 扩展到 10-20 道核心题，覆盖 3-4 个知识点
- 接入用户系统（手机号登录）+ 真实支付
- 增加学习记录与错题本

### V3（3-6 个月后）

- 引入 AI 对话引导（用户上传题目，AI 分步反问引导）
- 可视化知识图谱（展示题目间的关联）
- 多端适配（微信小程序版本）
