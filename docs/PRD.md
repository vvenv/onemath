# 一道+ / edao.plus 产品需求文档（MVP）

---

## 1. 项目概述

### 1.1 项目背景

- 小学奥数（数学思维训练）市场规模达 48 亿元，年复合增长率 12.3%
- 现有产品多为“大而全”的静态题库，缺乏“深入浅出”的动态讲解能力
- 家长和学生的核心痛点：**“题目有答案，但看不懂推导过程”**

### 1.2 产品定位

产品名：**一道+ / edao.plus**。一款 **以高质量讲解为核心** 的轻量级小学奥数思维训练工具。不做大而全的题库堆砌，而是通过 **分步图文 + 多解法对照 + 结构化可视化场景 + 方法手册互链 + （后续）AI 引导** 的组合，让每一道题都“有人带着想清楚”。

### 1.3 目标用户

| 用户类型        | 核心诉求                                     |
| :-------------- | :------------------------------------------- |
| 3-6 年级小学生  | 有奥数兴趣或培优需求，需要清晰易懂的解题思路 |
| 陪读家长        | 希望有能力辅导孩子，但自身对奥数解法不熟悉   |
| 独立老师/小机构 | 需要高质量的讲解素材辅助教学                 |

| :----- | :--------------------------------------- |
| MVP    | Minimum Viable Product，最小可行产品     |
| 变式题 | 与母题知识点相同、情境或数据不同的衍生题 |

## 2. MVP 范围

### 2.1 MVP 目标

跑通“价值感知 → 持续回访”的完整链路，验证用户是否认可“讲得清楚”这件事，并据此决定后续是否进一步投入内容生产与付费能力。

### 2.2 MVP 包含功能

| 模块       | 功能点                                                                      | 优先级 | 说明                                                                    |
| :--------- | :-------------------------------------------------------------------------- | :----- | :---------------------------------------------------------------------- |
| 题库首页   | 按 **模块 / 年级 / 难度 / 方法标签** 组合筛选，年级分组可折叠，筛选态持久化 | P0     | `@/pages/home-page.tsx`；筛选通过 `sessionStorage` 保存                 |
| 全局搜索   | App Bar 内的全局搜索（`@/components/global-search.tsx`）                    | P0     | 支持按 id / 标题 / 标签模糊匹配                                         |
| 单题页     | 题干 + 可选 SVG 图示 + 多解法图文 + 变式练习 + 知识点 + 关联推荐            | P0     | `@/pages/problem-page.tsx`                                              |
| 图文解析   | 多解法分步拆解 + 结构化可视化场景（`@/components/visuals/*`）               | P0     | 所有解法完整展示，不做解锁墙                                            |
| 举一反三   | 结构化输入的变式练习 + 正误判断 + 提示 + 可选图示                           | P0     | `@/components/practice-card.tsx`                                        |
| 方法手册   | 方法 / 模型知识点讲解独立路由（`/k`、`/k/:slug`）                           | P0     | 含直观场景、推导、要点、典型例题、常见误区、关联知识点                  |
| 知识点入口 | 单题页底部的「知识点」区，已收录的方法直接跳转手册                          | P0     | 未收录的方法保留占位态，收集需求信号（`@/components/knowledge-points`） |
| 关联推荐   | 同类题目推荐（按标签重合 + 同模块 + 同年级加权打分）                        | P0     | `getRelatedProblems` in `@/lib/problems.ts`                             |
| 方法标签   | 方法 / 模型标签（不含年级 / 难度 / 模块）                                   | P0     | 白名单见 `@/lib/tags.ts`，手册与题目共用同一词表                        |
| 主题与 PWA | 浅色 / 深色主题切换、PWA 可安装、离线 Service Worker                        | P1     | `@/lib/theme.ts`、`public/manifest.webmanifest`、`sw.js`                |
| 产品分析   | Vercel Analytics + Speed Insights                                           | P1     | 注入在 `@/root.tsx`，用于观测核心验证指标                               |

### 2.3 MVP 不包含功能

- **视频讲解**（V2 再做，MVP 阶段重资产 ROI 过低）
- 用户登录/注册系统
- 任何付费 / 留资 / 解锁墙相关流程（MVP 完全免费开放）
- 题库后台管理系统（题目以 TS 模块形式直接入库，由 JSON Schema 校验结构）
- AI 对话引导（V3 规划）
- 学习进度追踪与数据看板
- 收藏 / 历史记录等个性化能力（需配合用户系统）

## 3. 详细功能设计

### 3.1 题目信息模块

- **内容**：题干文字 + 关键数据高亮
- **示例**：
  > 笼子里有鸡和兔，共 **30个头**，**88只脚**。问鸡和兔各有多少只？

### 3.2 图文解析模块

同一道题提供多种解法分步拆解，全部默认可见。每个解法由若干 `steps`（文字）与可选的 `scenes`（结构化可视化片段，如比较条、头脚图、方程列表等）组成，做到“读文字 + 看图示”同步推进。

下面以“鸡兔同笼”为例展示三种解法的节奏：

**方法一：画图法**

- ① 画 30 个圆圈代表 30 个头

  ```
  ○ ○ ○ ○ ○ ○ ○ ○ ○ ○
  ○ ○ ○ ○ ○ ○ ○ ○ ○ ○
  ○ ○ ○ ○ ○ ○ ○ ○ ○ ○
  ```

- ② 每个下面画 2 只脚（假设全是鸡）→ 共 60 只脚

  ```
  ○ ○ ○ ○ ○ ○ ○ ○ ○ ○   ← 30 个头
  ∥ ∥ ∥ ∥ ∥ ∥ ∥ ∥ ∥ ∥   ← 每个 2 只脚
  ∥ ∥ ∥ ∥ ∥ ∥ ∥ ∥ ∥ ∥   合计：30 × 2 = 60 只脚
  ∥ ∥ ∥ ∥ ∥ ∥ ∥ ∥ ∥ ∥
  ```

- ③ 实际 88 只，还差 28 只

  ```
  目标  ████████████████████████████████████████████  88
  当前  ████████████████████████████████              60
                                        ↑ 还差 28 只脚
  ```

- ④ 每只鸡补 2 只脚变兔 → 补 14 只（28 ÷ 2 = 14）

  ```
  ○ ○ ○ ○ ○ ○ ○ ○ ○ ○ ○ ○ ○ ○ │ ○ ○ ○ ○ ○ ○ ○ ○ ○ ○ ○ ○ ○ ○ ○ ○
  ∥ ∥ ∥ ∥ ∥ ∥ ∥ ∥ ∥ ∥ ∥ ∥ ∥ ∥ │ ∥ ∥ ∥ ∥ ∥ ∥ ∥ ∥ ∥ ∥ ∥ ∥ ∥ ∥ ∥ ∥
  ╳ ╳ ╳ ╳ ╳ ╳ ╳ ╳ ╳ ╳ ╳ ╳ ╳ ╳ │
  ╳ ╳ ╳ ╳ ╳ ╳ ╳ ╳ ╳ ╳ ╳ ╳ ╳ ╳ │
       🐰 兔（4 脚）× 14        │        🐔 鸡（2 脚）× 16
  ```

- ⑤ 结论：兔 14 只，鸡 16 只

  ```
  🐰×14 + 🐔×16 = 30 只（头）
  4×14 + 2×16 = 56 + 32 = 88 只（脚）✓
  ```

**方法二：假设法**

- ① 假设全是鸡：30 × 2 = 60 只脚
- ② 实际相差：88 - 60 = 28 只脚
- ③ 每只兔比鸡多 2 只脚
- ④ 兔数：28 ÷ 2 = 14 只
- ⑤ 鸡数：30 - 14 = 16 只

**方法三：方程法（高年级适用）**

- ① 设鸡 x 只，兔 y 只
- ② x + y = 30
- ③ 2x + 4y = 88
- ④ 解得：x = 16，y = 14

### 3.3 举一反三模块

- **变式题目**：
  > 停车场上共有汽车和摩托车 28 辆，轮子总数 86 个。汽车 4 轮，摩托车 2 轮，问各有多少辆？
- **交互**：结构化字段输入（单/多字段，支持 `number` / `text` 等类型），提交后逐字段显示正误判断，错误时可展开 `hint`
- **可选图示**：变式题可附带独立的 `figures`（SVG）
- **可见性**：与主解法一致，默认完全开放

### 3.4 知识点标签模块

- 展示本题关联的**解题方法 / 模型标签**（取自 `@/lib/tags.ts` 的白名单，如 `鸟头模型` `假设法` `容斥原理`）
- 年级 / 模块 / 难度不进 `tags`，它们已经是独立字段（见 `problem-header` 的过滤逻辑）
- 标签分组：`counting`、`word`、`numberCalc`、`geometry`、`magicSquare`、`general`
- 首页「更多筛选」可按标签过滤；手册条目的 `tag` 字段与白名单对齐，实现题目 ↔ 手册双向跳转

### 3.5 方法手册（相关知识点讲解）

- **位置**：题目解析模块之后、练习模块之前，以「知识点」小节呈现
- **内容来源**：从题目 `tags` 中逐个按白名单查找对应的 `KnowledgeEntry`（通过知识页的 `tag` 字段匹配），命中则生成跳转条目
- **入口形态**：
  - **已收录** 的方法：条目直接跳转到 `/k/:slug` 手册页
  - **未收录** 的方法：保留可展开的占位条目，展示“讲解内容正在整理中”文案，继续作为需求信号探针
- **手册页结构**（`@/pages/knowledge-page.tsx`）：
  - Header：分类 Badge + 方法名 + 一句摘要
  - **直观场景**：一句话描述 + 可选 SVG 图示（`figures`）
  - **推导思路**：有序步骤，可附「公式 / 要点」折叠块
  - **典型例题**：2–3 道递进例题，每题含 `problem` / `solution` 步骤 / 可选 `takeaway`
  - **常见误区**：可选的 `pitfalls` 列表
  - **用到此方法的题目**：基于标签自动汇总的题目卡片（上限 8 条）
  - **相关知识点**：通过 `relatedSlugs` 在手册内互链
- **手册索引页** `@/pages/knowledge-index-page.tsx`：按分类（应用题 / 计数 / 几何 / 计算·数论 / 幻方 / 通用思想）折叠展示全部条目

## 4. 技术方案

### 4.1 技术栈选型

| 层级 | 技术                                                         | 说明                                                     |
| :--- | :----------------------------------------------------------- | :------------------------------------------------------- |
| 框架 | React 19 + React Router v7（framework 模式）                 | 文件式路由、SSR/SSG 兼容，便于后续接真实后端             |
| 构建 | Vite 8 + pnpm                                                | 开发体验与产物质量兼顾                                   |
| UI   | TailwindCSS v4 + shadcn/ui + Radix Primitives + lucide-react | 组件约束见 `AGENTS.md`                                   |
| 数据 | TS 模块题库（`src/data/problems/*.ts`） + JSON Schema 校验   | 支持 `import.meta.glob` 热加载；SVG 通过 `?raw` 原文引入 |
| PWA  | `vite-plugin-pwa` + 自定义 Service Worker                    | 支持离线缓存与安装                                       |
| 观测 | `@vercel/analytics` + `@vercel/speed-insights`               | 对齐 5.3 节验证指标                                      |
| 部署 | Vercel                                                       | 免费，自动 CI/CD                                         |
| 后端 | 暂不需要                                                     | MVP 纯静态；题目以代码形式入库                           |

### 4.2 数据结构

#### 4.2.1 题目

以实际 schema 为准（详见 `@/data/problem.schema.json`）。每道题是一个 TS 模块，默认导出 `ProblemData`：

```ts
import svg1 from "./figures/10001-1.svg?raw";

export default {
  id: "10001",
  title: "鸡兔同笼·基础",
  grade: "四年级",
  module: "word", // calc / geometry / numberTheory / word / travel / counting / misc
  difficulty: "基础", // 基础 / 进阶 / 挑战
  question: "笼子里有鸡和兔，共 30 个头，88 只脚。问鸡和兔各有多少只？",
  figures: [{ svg: svg1 }],
  solutions: [
    {
      key: "drawing",
      label: "画图法",
      steps: ["步骤 1...", "步骤 2..."],
      scenes: [{ kind: "heads", heads: { count: 30 } }],
    },
    { key: "hypothesis", label: "假设法", steps: ["..."] },
    { key: "equation", label: "方程法", steps: ["..."] },
  ],
  variant: {
    question: "停车场上共有汽车和摩托车 28 辆...",
    fields: [
      { key: "car", label: "汽车", type: "number" },
      { key: "motorcycle", label: "摩托车", type: "number" },
    ],
    answer: { car: 15, motorcycle: 13 },
    hint: "提示：先假设全是摩托车…",
  },
  tags: ["假设法", "方程法"],
};
```

**可视化场景** `scenes[].kind` 目前支持：`heads` / `compareBars` / `equationList` / `cubeNet` / `numberLine` / `numberGrid` / `pitDiagram` / `statementTable` / `resultBadges` / `lattice` / `svgFigure`（见 `@/components/visuals`）。

#### 4.2.2 知识点（方法手册）

条目存放于 `src/data/knowledge/<category>/*.ts` 并在 `@/data/knowledge.ts` 汇总：

```ts
export type KnowledgeEntry = {
  slug: string; // 路由段：/k/:slug
  name: string; // 手册标题
  tag?: string; // 对齐 tags.ts 白名单，用于题目 ↔ 手册互链
  category:
    | "word"
    | "counting"
    | "geometry"
    | "numberCalc"
    | "magicSquare"
    | "general";
  summary: string;
  intuition: string;
  figures?: { svg: string }[];
  derivation: string[];
  keyPoints?: string[];
  examples: {
    title?: string;
    problem: string;
    solution: string[];
    takeaway?: string;
  }[];
  pitfalls?: string[];
  relatedSlugs?: string[];
};
```

### 4.3 路由与页面结构

路由定义见 `@/routes.ts`：

| 路径       | 页面                       | 说明                             |
| :--------- | :------------------------- | :------------------------------- |
| `/`        | `home-page.tsx`            | 题库首页 / 筛选 / 按年级折叠列表 |
| `/p/:id`   | `problem-page.tsx`         | 单题页                           |
| `/k`       | `knowledge-index-page.tsx` | 方法手册索引                     |
| `/k/:slug` | `knowledge-page.tsx`       | 单个方法手册                     |

**全局布局** `@/layout.tsx` 内含 `AppBar`（Logo、全局搜索、手册入口、主题切换）。

**单题页组件树**：

```
ProblemPage
├── ProblemHeader       返回、题号、标题、年级/模块 Badge、方法标签、随机一题
├── QuestionCard        题干 + 可选 SVG 图示
├── SolutionTabs        多解法图文解析（全部默认开放，Scene + 文字步骤）
├── Methods             按 `tags` 生成方法手册跳转条目
├── PracticeCard        举一反三练习（结构化输入 + 正误 + hint）
└── RelatedProblems     基于标签 / 模块 / 年级打分的推荐
```

## 5. 验收标准

### 5.1 功能验收

- [ ] 首页可按模块 / 年级 / 难度 / 方法标签组合筛选，刷新后筛选态保留
- [ ] 全局搜索可按 id / 标题 / 标签命中题目并跳转
- [ ] 题目信息正确展示（题干 + 可选图示）
- [ ] 多解法全部默认可见，文字步骤与 `scenes` 可视化对应正确
- [ ] 变式题输入答案后可判断正误，必要时展示提示
- [ ] 方法标签仅来自 `tags.ts` 白名单，不混入年级 / 模块 / 难度
- [ ] 已收录方法点击后跳转 `/k/:slug`；未收录方法展开占位文案
- [ ] 方法手册页直观场景、推导、要点、例题、误区、关联题、相关知识点均正确渲染
- [ ] 浅色 / 深色主题切换生效并持久化
- [ ] 页面在移动端与桌面端均可正常显示；PWA 可离线访问已访问路径

### 5.2 体验验收

- [ ] 图文解析步骤分明，易于跟随
- [ ] 可视化场景与文字步骤语义一致，SVG 适配深浅主题（`currentColor`）
- [ ] 首屏加载速度 < 2 秒（Vercel Speed Insights 90 分位）

### 5.3 产品验证指标

| 指标           | 目标值 | 说明                                         |
| :------------- | :----- | :------------------------------------------- |
| 解析读完率     | > 50%  | 滚动到解析底部 / 总访问                      |
| 多解法切换率   | > 30%  | 切换过至少一次解法 Tab / 总访问              |
| 方法手册点开率 | > 15%  | 展开任一知识点条目或进入 `/k/:slug` / 总访问 |
| 变式题尝试率   | > 40%  | 提交过一次变式答案 / 总访问                  |
| 关联题跳转率   | > 20%  | 点击关联题卡片 / 总访问                      |

## 6. 后续迭代规划

> MVP 已落地内容：~140 道题目 + 方法手册独立页（首批覆盖应用题 / 计数 / 几何 / 计算·数论 / 幻方 / 通用思想六类的核心方法）。

### V2（1-2 个月）

- **内容扩量**：持续扩充题目与模块覆盖；补齐方法手册剩余白名单条目（把单题页仍为「占位态」的方法逐一转正）
- **收藏与浏览历史**：基于 `localStorage` 的轻量个性化，无需账号
- **引入视频讲解**：3 分钟手绘风格，基于前期点开率决定自制 or 外包
- **标签页 / 方法聚合页**：点击标签进入同方法题目聚合视图
- 按需再评估用户系统与付费能力

### V3（3-6 个月）

- 引入 AI 对话引导（用户上传题目，AI 分步反问引导）
- 可视化知识图谱（题目 ↔ 方法 ↔ 题目 的网络视图）
- 多端适配（微信小程序版本）
